custom-* 컨벤션

표준 카탈로그에 없는 영역은 custom-${'{string}'} template literal type 으로 자유 등록 가능. plugin 끼리 협업/실험용.

권장 명명: custom-<pluginId>-<slot> — 예: custom-archive-profile-tooltip. 표준 slot 과 plugin 끼리 이름 충돌을 회피한다.

사용 예시

plugin.json

{
  "id": "archive",
  "components": [
    {
      "id": "archive-button",
      "name": "보존 버튼",
      "slot": "post-detail-extra",
      "path": "views/ArchiveButton.svelte",
      "priority": 50
    },
    {
      "id": "archive-profile-stat",
      "slot": "member-profile-stats",
      "path": "views/ArchiveProfileStat.svelte",
      "priority": 10
    }
  ]
}

core 측 마운트 (Sprint 2 에서 페이지 사전 배치)

<!-- routes/member/[id]/+page.svelte -->
<PluginSlot name="member-profile-stats" mbId={p.mb_id} />

plugin component 가 받는 props

<script lang="ts">
    let { mbId }: { mbId: string } = $props();
    // ...
</script>

SSR / Performance 주의

  • <PluginSlot> 은 client-only (registry 가 client singleton, onMount 후 갱신). hydration 후 plugin component 가 fade-in 된다.
  • Critical content 영역 (above-the-fold) 에 plugin component 마운트 부담 — 가급적 보조 영역에만 슬롯 배치.
  • slot 미사용 시 (components.length === 0) DOM 자체 미렌더 → 레이아웃 영향 0.
